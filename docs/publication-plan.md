# Publication Plan

## Formal Track

Target output:
- technical report or preprint-ready paper

Sections expected:
- abstract
- introduction
- related work
- benchmark selection rationale
- methodology
- baseline and Iranti conditions
- results
- threats to validity
- discussion
- conclusion

## Public Track

Target output:
- blog or journal-style explainer for technical generalists

Requirements:
- plain English
- clear statement of what was tested
- no overclaiming
- include what Iranti changed and what it did not change
- explain why the benchmark matters in practical terms

## Shared Rule

No public claim should appear in the public track unless the formal track can support it.
