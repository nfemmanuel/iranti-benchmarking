## Objective

Design and execute Benchmark B1: an NIAH-inspired evaluation of entity-specific fact retrieval accuracy under increasing distractor density.

This benchmark tests whether Iranti's exact entity/key retrieval (`iranti_query`) maintains higher precision than document-reading (baseline) as the number of distractor entities increases.

---

## Background

The Needle-in-a-Haystack (NIAH) benchmark (Kamradt, 2023) tests whether an LLM can retrieve a specific piece of information from a long context document. We adapt this to structured entity/key retrieval:

- The "haystack" is a document listing N entities × 4 facts each
- The "needle" is a specific fact about a specific entity
- We vary N (distractor density) across conditions
- We add an adversarial condition where wrong facts about the target entities also appear

---

## Scope

**In scope:**
- Baseline arm: Claude Sonnet 4.6 reading from context document
- Iranti retrieval arm: `iranti_query` for exact entity/key lookup on existing KB data
- 3 distractor density conditions: N=5, N=20, N=20+adversarial

**Out of scope:**
- Iranti write arm (ingest → retrieve cycle): blocked by infrastructure issue (see BM-T004)
- Statistical significance testing (n too small for this session; noted as future work)

## Threats to Validity

1. **Self-evaluation bias**: The model serving as baseline (Claude Sonnet 4.6) is the same model running this evaluation. Results may reflect self-consistency rather than independent performance.
2. **Write arm unavailable**: Cannot test the full ingest-then-retrieve cycle due to DB connection pool exhaustion.
3. **Synthetic dataset**: Ground truth is constructed, not drawn from a validated public benchmark corpus.
4. **Small n**: 30 baseline data points is insufficient for strong statistical claims. Results are indicative.
5. **Single model**: Baseline uses only Claude Sonnet 4.6. Other models may show different patterns.
