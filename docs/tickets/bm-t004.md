## Problem

During the 2026-03-20 benchmarking session, all Iranti write operations failed:

```
Error: Invalid `client_1.getDb)().knowledgeEntry.findUnique()` invocation
Too many database connections opened: sorry, too many clients already
```

The error appears in `iranti/dist/src/library/agent-registry.js:83` (the `updateStats` function called during writes), and also in `iranti/dist/src/attendant/...` during handshake (relationships query).

Read operations (`iranti_query`, `iranti_search`) are **fully functional**.

## Steps to Resolve

1. **Restart Iranti instance:**
   ```powershell
   # Kill current instance
   # In a new terminal:
   iranti run --instance local
   ```

2. **Verify write capability:**
   ```powershell
   iranti doctor --instance local
   iranti status
   ```

3. **Test via MCP:**
   - Run `iranti_write` with a test entity/key
   - Confirm it succeeds without connection error

4. **Investigate connection pool settings:**
   - Check `DATABASE_URL` in `.env.iranti` for `pool_max` or connection pool parameters
   - PostgreSQL default max_connections is typically 100; check if this is hit
   - Consider setting a connection pool limit in the Prisma/pg adapter config
