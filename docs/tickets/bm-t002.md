## Objective

Design and execute Benchmark B2: a test of whether facts established in one session are correctly recalled in a fresh session with no context carryover.

This benchmark directly tests Iranti's core value proposition: **persistent memory that survives session boundaries**.

---

## Benchmark Design

### Task Definition

An agent learns N facts about M entities in Session 1. In Session 2 (fresh context, no prior conversation), the agent is asked to recall those facts. Accuracy is measured.

### Arms

**Baseline arm:**
- Session 1: Facts provided in conversation context
- Session 2: New conversation, no context provided, no Iranti
- Expected result: 0% recall (no persistence mechanism)

**Iranti arm:**
- Session 1: Facts written to Iranti KB via `iranti_write`
- Session 2: New conversation, no context provided; agent uses `iranti_query` to retrieve
- Expected result: High recall (facts persisted in KB)

### Metrics

- Recall rate (proportion of facts correctly stated in Session 2)
- Precision (no false facts asserted)
- Entity disambiguation accuracy (correct entity matched to correct facts when multiple entities exist)

### Dataset

- 5 target entities × 4 facts = 20 facts
- Entities will be fictional researchers with distinct profiles
- Questions will probe specific entity/key combinations

## Threats to Validity

1. **Baseline is definitionally 0%**: The baseline cannot recall facts without context. This is expected, not a finding. The claim is that Iranti enables recall, not that the baseline is deficient.
2. **Write arm dependency**: Without write capability, cannot demonstrate the full Session 1 → persist → Session 2 cycle.
3. **Single-session execution**: In production, Session 1 and Session 2 would be separate invocations. In testing, they're sequential steps in one evaluation. The reconvene mechanism should still be testable.
