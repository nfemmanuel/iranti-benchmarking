# Benchmark Evidence Discipline

## Purpose
Keep the benchmark program conservative, reproducible, and externally defensible.

## Rules
1. Use the installed Iranti product surface, not the local source checkout.
2. Treat the installed CLI/runtime as the oracle for product behavior.
3. Audit first when rerun readiness is unclear.
4. Separate valid claims from unsupported claims in every major report.
5. Do not hide null, weak, or blocked tracks.
6. If a task only demonstrates a structural limit, describe it as a structural result, not a stronger measured-comparison result.

## Required Report Shape
1. what was reviewed or tested
2. what is valid already
3. what remains blocked or weak
4. what claims are still invalid
5. the smallest safe next decision
