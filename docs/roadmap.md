# Benchmark Roadmap

## Phase 1 - Reset And Research Infrastructure

- establish fresh repo purpose and operating rules
- create research team agent scaffolding
- bind workspace to installed Iranti instance
- define benchmark selection criteria
- define publication outputs

## Phase 2 - Benchmark Discovery And Triage

- survey standard or recognizable benchmark families
- shortlist credible replication candidates
- document inclusion and exclusion rationale
- identify what can be replicated faithfully versus only approximated

## Phase 3 - Experimental Design

- define baseline arm for each chosen benchmark
- define Iranti-assisted arm for each chosen benchmark
- define metrics, controls, and threats to validity
- define result artifact format

## Phase 4 - Replication And Execution

- implement benchmark tracks
- run baseline trials
- run Iranti-assisted trials
- collect raw results and logs
- review reproducibility

## Phase 5 - Analysis And Review

- statistical review
- failure analysis
- replication audit
- claim-hardening review

## Phase 6 - Publication

- formal scientific paper/report
- appendix and methodology notes
- public blog/journal article
